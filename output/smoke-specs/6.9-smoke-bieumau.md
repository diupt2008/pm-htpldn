## Ghi chú đặc thù module
- **Precondition data:** ≥1 lĩnh vực PL trong `DANH_MUC` (FK `linh_vuc_id`). Nếu filter rỗng → báo dev seed.
- **AntD Upload (nếu test upload ở Bước 3):** dùng `$B setFiles` với selector `input[type="file"]`, **tránh** drag-drop.
- Badge trạng thái qua `.ant-tag` — verify bằng text (`NHAP`/`CONG_KHAI`/`AN`), không theo color class.

## Bước 2: VÀO MODULE

### 2a. Verify menu + (submenu nếu có) sidebar

1. `$B snapshot -i`
2. Grep `Quản lý thư viện biểu mẫu` hoặc `Biểu mẫu`
3. Nếu có submenu: click menu → đợi expand → grep `Thư viện biểu mẫu`

| Kết quả | Action |
|---------|--------|
| ✅ Menu visible (+ submenu nếu có) | → 2b |
| ❌ Menu không thấy | Verify role. Thiếu → **FAIL**, báo FE |
| ❌ Submenu không expand | Retry 1 lần. Vẫn không → **FAIL** |

### 2b. Navigate

1. Click submenu đích (hoặc menu trực tiếp)
2. Đợi page render (< 5s)
3. Verify URL + tabs ≥3/4 + nút `+ Thêm thư mục` + filter lĩnh vực
4. Chụp `bieumau-page.png`

| Kết quả | Action |
|---------|--------|
| ✅ URL + elements | → Bước 3 |
| ❌ Redirect `/login` | Session lỗi, báo dev |
| ❌ Page trắng | **FAIL**, báo BE |
| ❌ Thiếu tabs/buttons | **FAIL**, note |

## Bước 3: KIỂM TRA LỖI NGẦM

1. `$B console --errors`
2. `$B network` → grep 4xx/5xx
3. `$B snapshot` → grep `"Lỗi"`, `"Validation failed"`, `"Không thành công"`, `"undefined"`

| Kết quả | Verdict |
|---------|---------|
| Sạch | ✅ PASS |
| Warning / 401 single | ⚠️ WARN |
| TypeError / 5xx / toast đỏ | ❌ FAIL |
