## Ghi chú đặc thù module
- **Precondition data:** Cần ≥1 DN + ≥1 NHT (Người hưởng thụ) + ≥1 lĩnh vực. Nếu thiếu, form tạo không submit được (ngoài scope smoke).
- **Yêu cầu bổ sung (YEU_CAU_BO_SUNG):** tối đa 3 lần. Detail view có counter — verify không bị âm hoặc overflow.
- **Known UX issue:** list có nhiều column → có thể cần horizontal scroll. Không block verdict nếu scroll hoạt động.

- [ ] Server up: `curl -o /dev/null -w "%{http_code}" http://103.172.236.130:3000/` → `200`
- [ ] Account chưa lock:
  ```bash
  curl -s -X POST http://103.172.236.130:3000/api/v1/auth/login \
    -H "Content-Type: application/json" \
    -d '{"username":"canbo_tw","password":"Secret@123"}'
  ```

  | Response | Action |
  |----------|--------|
  | `"otpToken"` | → Bước 1 |
  | `"tạm khóa"` | **STOP**, đổi account |
  | `"Invalid credentials"` | **STOP**, verify password |

## Bước 2: VÀO MODULE

### 2a. Verify menu sidebar

`$B snapshot -i` + grep `Quản lý vụ việc hỗ trợ pháp lý`.

| Kết quả | Action |
|---------|--------|
| ✅ Menu visible | → 2b |
| ❌ Không thấy | Verify role. Thiếu → **FAIL**, báo FE |

*(Không có submenu.)*

### 2b. Navigate

1. Click menu
2. Đợi page render (< 5s)
3. Verify URL + tabs 12 state + nút `+ Tạo vụ việc`
4. Chụp `vuviec-page.png`

| Kết quả | Action |
|---------|--------|
| ✅ URL + elements | → Bước 3 |
| ❌ Redirect `/login` | Session lỗi, báo dev |
| ❌ Page trắng | **FAIL** |
| ❌ Thiếu tabs | **FAIL**, note |

## Bước 3: KIỂM TRA LỖI NGẦM

1. `$B console --errors`
2. `$B network` → grep 4xx/5xx
3. `$B snapshot` → grep lỗi toast

| Kết quả | Verdict |
|---------|---------|
| Sạch | ✅ PASS |
| Warning / 401 | ⚠️ WARN |
| TypeError / 5xx / toast đỏ | ❌ FAIL |

## Verdict tổng

| Verdict | Điều kiện | Unlock? |
|---------|-----------|---------|
| ✅ PASS | 4/4 PASS, không 5xx | Yes |
| ⚠️ WARN | 4/4 + warning | Yes |
| ❌ FAIL | ≥1 FAIL | No |
| 🔒 BLOCKED | Crash 2/lock/down | Skip |
