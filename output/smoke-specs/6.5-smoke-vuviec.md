## Ghi chú đặc thù module

- **Precondition data:** Cần ≥1 DN + ≥1 cá nhân được phân công (TVV/CG state HOAT_DONG có TAI_KHOAN linked, HOẶC NHT entity riêng `NGUOI_HO_TRO`) + ≥1 lĩnh vực PL. Nếu thiếu form tạo không submit được (ngoài scope smoke).
- **Phân công xử lý (FR-V.I-09 refactor):** Modal **2 thẻ** Cá nhân/Tổ chức. Bỏ field `nguoi_ho_tro_id` cũ. KHÔNG còn dropdown "Địa bàn" (NĐ 77/2008 Đ.19).
- **Yêu cầu bổ sung (YEU_CAU_BO_SUNG):** Tối đa 3 lần (BR-EC-15). DN bổ sung qua **SCR-V.I-04 chuyên trang DN** (FR-V.I-NEW-02 — auth Tier 2 VNeID), KHÔNG còn qua DVC inbound như v3.
- **Công khai cờ overlay (NEW 2026-05-06):** Trên SCR-V.I-03 khi VV ở `DA_DUYET` hoặc `HOAN_THANH` xuất hiện Badge xanh "Đã công khai" (khi `cong_khai=1`) + 2 nút `[Công khai]` / `[Hủy công khai]` chỉ hiển thị cho **CB PD cùng cấp** (BR-AUTH-05).
- **Migration risk:** Record VV cũ có `nguoi_ho_tro_id` — SRS update không cover migration → khi gặp record có field này, **log bug data migration**, hỏi BA.
- **Known UX issue:** List có nhiều cột → có thể cần horizontal scroll. Không block verdict nếu scroll hoạt động.

1. `click({uid: menu_uid})`
2. `wait_for({text: ["Danh sách vụ việc", "Tổng cộng"], timeout: 15000})` *(signal page render)*

4. Verify **12 tabs state** + nút `[+ Tạo vụ việc]` + cột "Công khai" trong bảng list (NEW)

| Kết quả | Action |
|---------|--------|
| ✅ URL + 12 tabs + cột Công khai | → Bước 3 |
| ❌ Redirect `/login` | Session lỗi, báo dev (KHÔNG dùng `navigate_page` sau login — MCP-Rule 3) |

## Smoke Optional (mở rộng FR-05 v3.5 — chạy khi đủ data + tool sẵn sàng)

### TP-Optional-1: Smoke flow Công khai VV (FR-V.I-NEW-05)

**Precondition:** ≥1 VV ở state `DA_DUYET` hoặc `HOAN_THANH`. Cần đăng nhập **CB PD cùng cấp** với CB NV xử lý VV (BR-AUTH-05).

1. Login `cb_pd_tw_01` (per Bước 1).
2. Mở danh sách VV → tab "Hoàn thành" → click VV target → mở SCR-V.I-03 chi tiết.
3. Verify nút **[Công khai]** xuất hiện (không có ở role CB NV).
4. Click [Công khai] → form mở: nhập `mo_ta_cong_khai` ≤2000 ký tự + upload `anh_dai_dien` jpg/png ≤5MB + chọn `file_dinh_kem_cong_khai` ≤10 file.
5. `mcp__chrome-devtools__list_network_requests` track API outbound đến Cổng PLQG.
6. **Verify response shape API outbound** — chỉ chứa **9 fields whitelist** (BR-PUBLIC-04):
   - `linh_vuc_phap_luat`, `loai_hinh_ho_tro`, `mo_ta_cong_khai`, `thoi_gian_xu_ly`, `don_vi`, `ket_qua`, `thoi_gian_dang_tai`, `anh_dai_dien`, `file_dinh_kem_cong_khai`
   - **TUYỆT ĐỐI KHÔNG có:** tên DN/người đại diện, CCCD/MST, mo_ta nội bộ, file_dinh_kem nghiệp vụ, noi_dung_tu_van, SĐT/email/địa chỉ DN.
7. Verify Badge xanh "Đã công khai" hiển thị + tooltip ngày dd/mm/yyyy.

| Kết quả | Verdict |
|---------|---------|
| API outbound ≤9 fields whitelist + Badge xanh | ✅ PASS |
| Leak ≥1 field nhạy cảm (đặc biệt **tên DN/MST**) | ❌ **FAIL Critical** — escalate ngay (privacy breach NĐ 13/2023) |
| API trả 5xx hoặc set `cong_khai=1` trước khi API OK | ❌ FAIL — vi phạm BR-EC-20 |

### TP-Optional-2: Smoke flow DN bổ sung HS (FR-V.I-NEW-02)

**Precondition:** ≥1 VV ở state `YEU_CAU_BO_SUNG` thuộc DN sẽ login. TK DN VNeID Tier 2 sandbox.

1. Logout CB. Login DN qua VNeID Tier 2 (per memory `project_auth_no_vnpt_ekyc` — KHÔNG VNPT eKYC).
2. Mở SCR-V.I-04 "Vụ việc của tôi" → tab "Đang xử lý" → click VV có **badge "Cần bổ sung"**.
3. Verify SCR-V.I-04 ẨN tên cá nhân CB (chỉ hiển thị "Đơn vị xử lý" — NĐ 13/2023).
4. Click nút [Bổ sung hồ sơ] → form mở: upload `file_bo_sung[]` (PDF/DOC/JPG/PNG ≤20MB/file, tổng ≤100MB, max 10 file) + `ghi_chu` ≤5000 ký tự.
5. Click [Gửi] → verify state VV chuyển `YEU_CAU_BO_SUNG → DANG_KIEM_TRA`.
6. `list_network_requests` → verify POST endpoint `/api/v1/vu-viec/{id}/bo-sung` trả 200.

| Kết quả | Verdict |
|---------|---------|
| State chuyển + file upload OK | ✅ PASS |
| Form thấy tên cá nhân CB phụ trách | ❌ FAIL — vi phạm NĐ 13/2023 |
| ERR-VV-BS-04 "Bạn không có quyền" khi DN đúng chủ sở hữu | ❌ FAIL — BE bug auth |
| Sandbox VNeID không có | 🔒 SKIP — log workaround, escalate BA |

### TP-Optional-3: Smoke filter Công khai trên list

1. Vào danh sách VV (Bước 2b).
2. Verify cột "Công khai" trong bảng — record có `cong_khai=1` hiện badge xanh "Đã công khai".
3. Apply filter "Công khai" → verify chỉ list record có `cong_khai=1`.
