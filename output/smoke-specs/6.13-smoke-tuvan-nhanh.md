## Ghi chú đặc thù module
- **Permission:** CB NV (TW/BN/ĐP) có Create kho Q&A thủ công + trả lời phiên. CB PD duyệt Q&A (đơn lẻ + hàng loạt). DN = 🔌 API only (không vào CMS).
- **Cùng menu cha với TVCS:** Menu `Quản lý tư vấn ▶` chứa cả `Tư vấn chuyên sâu` (smoke 6.12) lẫn `Tư vấn nhanh` — đừng click nhầm submenu.
- **Precondition data:**
  - Không bắt buộc cho smoke 4 bước (chỉ verify trang load). Bước 2b có thể hiển thị list rỗng nếu DB sạch — vẫn PASS miễn header cột + tabs + nút + filter render đủ.
  - Nếu cần test gợi ý (Lệnh 4): seed ≥3 KHO_CAU_HOI DA_DUYET + hiệu_luc=true (xem [7.13 §Data Readiness](../funtion/7.13-tu-van-nhanh.md#data-readiness-cho-module-này)).
- **Rich Text Editor** cho trường câu trả lời — nếu test nâng cao, dùng `$B type` trên `[contenteditable="true"]` hoặc `.ql-editor`.
- **Known trap:** Form tạo Q&A và form soạn trả lời đều có textarea. Đừng nhầm — câu hỏi là `<textarea>` plain, câu trả lời là Rich Text.

## Bước 2: VÀO MODULE

### 2a. Verify menu + submenu sidebar

1. `$B snapshot -i`
2. Grep `Quản lý tư vấn` (có ▶) — phải xuất hiện
3. Click menu → đợi submenu expand
4. Snapshot lại → grep `Tư vấn nhanh` (không phải `Tư vấn chuyên sâu`)

| Kết quả | Action |
|---------|--------|
| ✅ Menu + submenu `Tư vấn nhanh` visible | → 2b |
| ❌ Menu không thấy | Verify role `canbo_tw` có quyền module `TVNHANH`. Thiếu → **FAIL**, báo FE |
| ❌ Submenu không expand / chỉ có `Tư vấn chuyên sâu` | Retry 1 lần. Vẫn thiếu → **FAIL**, báo FE (module TVN chưa deploy / chưa bật menu) |

### 2b. Navigate

1. Click submenu `Tư vấn nhanh`
2. Đợi page render (< 5s)
3. Verify:
   - URL route khớp expect
   - 2 tab chính `Kho câu hỏi` + `Phiên tư vấn` hiển thị
   - Tab `Kho câu hỏi` (default active) có 3 sub-tab `Tất cả / Đã duyệt / Chờ duyệt` + nút `+ Thêm câu hỏi`, `Nhập Excel`, `Làm mới`
   - Filter bar: Lĩnh vực / Nguồn / Trạng thái
   - Table header: Mã / Câu hỏi / Câu trả lời / Lĩnh vực / Nguồn / Trạng thái / Hiệu lực / Điểm TB / Ngày tạo / Hành động
4. Click sang tab `Phiên tư vấn` → verify 4 sub-tab `Tất cả / Chờ xử lý / Đã gợi ý / Hoàn thành` + header cột `Mã phiên / Câu hỏi DN / Kênh / Số gợi ý / Trạng thái / Ngày gửi`
5. Chụp `tvnhanh-kho-page.png` + `tvnhanh-phien-page.png`

| Kết quả | Action |
|---------|--------|
| ✅ URL + cả 2 tab + elements đầy đủ | → Bước 3 |
| ❌ Redirect `/login` | Session lỗi, báo dev |
| ❌ Page trắng / spinner kéo dài >10s | **FAIL**, báo BE |
| ❌ Chỉ có 1 tab (thiếu `Phiên tư vấn` hoặc thiếu `Kho câu hỏi`) | **FAIL**, note — có thể FE chưa gộp 2 tab theo spec v2.1 |
| ❌ Thiếu nút `+ Thêm câu hỏi` / `Nhập Excel` / filter | **FAIL**, note chi tiết |

## Bước 3: KIỂM TRA LỖI NGẦM

1. `$B console --errors`
2. `$B network` → grep 4xx/5xx (đặc biệt chú ý endpoint `/api/v1/kho-cau-hoi`, `/api/v1/tu-van-nhanh`)
3. `$B snapshot` → grep `"Lỗi"`, `"Validation failed"`, `"Không thành công"`, `"Có lỗi xảy ra"`, `"undefined"`

| Kết quả | Verdict |
|---------|---------|
| Sạch | ✅ PASS |
| Warning / 401 single (auth timing) | ⚠️ WARN |
| TypeError / 5xx / toast đỏ | ❌ FAIL |
