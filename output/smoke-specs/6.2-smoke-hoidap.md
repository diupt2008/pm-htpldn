## Pre-check (trước khi mở browse)

- [ ] **Server up:**
  ```bash
  curl -o /dev/null -w "%{http_code}" http://103.172.236.130:3000/
  ```
  Kỳ vọng: `200`. Không thì **STOP**, báo DevOps.

- [ ] **Account chưa bị khóa** — test qua API:
  ```bash
  curl -s -X POST http://103.172.236.130:3000/api/v1/auth/login \
    -H "Content-Type: application/json" \
    -d '{"username":"canbo_tw","password":"Secret@123"}'
  ```

  | Response chứa | Action |
  |---------------|--------|
  | `"otpToken"` | → Bước 1 |
  | `"tạm khóa"` | **STOP**, đổi account (xem [test-accounts.csv](../../input/test-accounts.csv)) |
  | `"Invalid credentials"` | **STOP**, verify password |

## Bước 1: LOGIN

1. Mở browser → `http://103.172.236.130:3000/`
2. Nhập `canbo_tw` / `Secret@123`
3. Submit → OTP page → nhập `666666`
4. Đợi dashboard (< 5s)
5. Chụp `hoidap-login-dashboard.png`

| Kết quả | Action |
|---------|--------|
| ✅ Dashboard + sidebar hiện | → Bước 2 |
| ❌ "Tài khoản tạm khóa" | **STOP**. BLOCKED, đổi account |
| ❌ "Tên đăng nhập hoặc mật khẩu không đúng" | **STOP**. Verify password |
| ❌ Page trắng / server 500 | **STOP**. Báo DevOps |

## Bước 2: VÀO MODULE

### 2a. Verify menu trong sidebar

1. `$B snapshot -i` sidebar
2. Grep label `Quản lý hỏi đáp pháp lý` — phải xuất hiện

| Kết quả | Action |
|---------|--------|
| ✅ Menu visible | → 2b |
| ❌ Không thấy menu | Verify role account có permission không. Role đúng mà thiếu → **FAIL**, báo FE |

*(Module này không có submenu — bỏ qua check submenu.)*

### 2b. Navigate vào module

1. Click menu `Quản lý hỏi đáp pháp lý` (hoặc goto URL nếu đã biết)
2. Đợi page render (< 5s)
3. Verify URL khớp expect + tabs/nút "+ Tạo" xuất hiện
4. Chụp `hoidap-page.png`

| Kết quả | Action |
|---------|--------|
| ✅ URL + elements đủ | → Bước 3 |
| ❌ Redirect `/login` | Session lỗi, báo dev |
| ❌ Page trắng >10s | **FAIL**, báo BE |
| ❌ Thiếu tabs/nút Tạo | **FAIL**, note chi tiết |

## Bước 3: KIỂM TRA LỖI NGẦM

1. `$B console --errors`
2. `$B network` → grep 4xx/5xx
3. `$B snapshot` → grep: `"Lỗi"`, `"Validation failed"`, `"Không thành công"`, `"Có lỗi xảy ra"`, `"undefined"`

| Kết quả | Verdict |
|---------|---------|
| Sạch | ✅ PASS |
| Warning deprecation / 401 single | ⚠️ WARN |
| `TypeError`/`undefined` / ≥1 lỗi 5xx / toast đỏ rõ ràng | ❌ FAIL |

## Crash / Retry Rule

Browse crash (`about:blank` / `Target page has been closed` / `timed out`) — xem [CLAUDE.md Rule 7](../../CLAUDE.md):

- **Lần 1:** cleanup + retry 1 lần
- **Lần 2:** **STOP ngay**, mark BLOCKED, báo user. Không retry lần 3.

## Verdict tổng

| Verdict | Điều kiện | Unlock? |
|---------|-----------|---------|
| ✅ PASS | 4/4 PASS, 0 lỗi 5xx | Yes — functional |
| ⚠️ WARN | 4/4 PASS + warning | Yes — note |
| ❌ FAIL | ≥1 bước FAIL | No — báo dev |
| 🔒 BLOCKED | Crash 2 lần / lock / down | Skip |
