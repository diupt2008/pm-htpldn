## Ghi chú đặc thù module (v3.5 update)
- **Module rename:** "Kế hoạch đánh giá" → "Theo dõi Đánh giá Hiệu quả Hỗ trợ Pháp lý" (Thay đổi 1, A-ITEM-08).
- **Permission:** Chỉ CB NV TW/BN/ĐP có Create. CB PD **KHÔNG** tạo được `KE_HOACH_DANH_GIA` (chỉ duyệt PC + BC). Dùng đúng `cb_nv_tw_01` smoke.
- **Precondition data:** Cần ≥1 lĩnh vực trong `DANH_MUC` + ≥1 đơn vị hợp lệ + ≥1 DON_VI khác cho `co_quan_duoc_danh_gia_id` 1:1. Nếu filter rỗng → precondition thiếu, note báo dev seed.
- **State 8 v3.5 + HUY:** Filter dropdown phải có đủ 8 state mới + transition `* → HUY`. Verify state cũ (`NHAP / DA_LAP_KH / DA_DUYET_PC / DANG_DANH_GIA / DA_DANH_GIA / DA_LAP_BC / CHO_DUYET_BC / DA_DUYET_BC`) **KHÔNG còn xuất hiện** — nếu thấy → log bug "BE chưa migrate enum SM-DANHGIA v3.5".
- **FR-VI-10 mới:** Sau khi đợt `HOAN_THANH`, CB NV thuộc `co_quan_duoc_danh_gia_id` xem được KQ read-only — smoke ngoài scope (test ở functional).
- Detail view có 4 tab (Tiêu chí / Phân công / Thực hiện chấm điểm / Báo cáo) — ngoài scope smoke.

## Bước 2: VÀO MODULE

### 2a. Verify menu + (submenu nếu có) sidebar

1. `mcp__chrome-devtools__take_snapshot`
2. Grep `Theo dõi Đánh giá Hiệu quả` hoặc `Đánh giá Hiệu quả HTPL`
3. Nếu có submenu: click menu → đợi expand → grep submenu `Theo dõi Đánh giá Hiệu quả HTPL`

| Kết quả | Action |
|---------|--------|
| ✅ Menu visible (+ submenu nếu có) | → 2b |
| ❌ Menu không thấy | Verify role. Thiếu → **FAIL**, báo FE |
| ❌ Submenu không expand | Retry 1 lần. Vẫn không → **FAIL** |

### 2b. Navigate

1. Click submenu đích (hoặc menu nếu không có submenu)
2. Đợi page render (< 5s)
3. Verify:
   - URL chứa `/danh-gia` hoặc `/ke-hoach-danh-gia`
   - Tiêu đề trang: "Theo dõi Đánh giá Hiệu quả HTPL"
   - Filter trạng thái dropdown có **đủ 8 state v3.5** (LAP_KE_HOACH/PHAN_CONG/CHO_DUYET_PC/THUC_HIEN/BAO_CAO/CHO_PHE_DUYET/HOAN_THANH/HUY)
   - Nút `+ Tạo đợt đánh giá` hiển thị
   - **State cũ (NHAP/DA_LAP_KH/...) KHÔNG xuất hiện**
4. Click `+ Tạo đợt đánh giá` → mở form → verify field `Cơ quan được đánh giá` (dropdown) + `File đính kèm` (upload zone)
5. Chụp `danhgia-page.png` + `danhgia-form-create.png`

| Kết quả | Action |
|---------|--------|
| ✅ URL + 8 state + form có 2 field mới | → Bước 3 |
| ❌ Redirect `/login` | Session lỗi, báo dev |
| ❌ Page trắng | **FAIL**, báo BE |
| ❌ Filter thiếu state mới hoặc còn state cũ | **FAIL** — log bug "SM-DANHGIA chưa migrate v3.5" |
| ❌ Form thiếu `co_quan_duoc_danh_gia_id` hoặc `file_dinh_kem` | **FAIL** — log bug "FR-08 v3.5 field chưa implement" |
| ❌ Tiêu đề vẫn là "Kế hoạch đánh giá" | ⚠️ WARN — log bug rename UI chưa apply |

## Bước 3: KIỂM TRA LỖI NGẦM

1. `mcp__chrome-devtools__list_console_messages({types:["error","warn"]})`
2. `mcp__chrome-devtools__list_network_requests({resourceTypes:["xhr","fetch"]})` → grep 4xx/5xx
3. `mcp__chrome-devtools__take_snapshot` → grep `"Lỗi"`, `"Validation failed"`, `"Không thành công"`, `"undefined"`

| Kết quả | Verdict |
|---------|---------|
| Sạch | ✅ PASS |
| Warning / 401 single | ⚠️ WARN |
| TypeError / 5xx / toast đỏ | ❌ FAIL |
