## Ghi chú đặc thù module
- **Permission:** Chỉ CB NV TW có Create. CB PD **KHÔNG** tạo được `KE_HOACH_DANH_GIA` (chỉ duyệt). Dùng đúng `canbo_tw`.
- **Precondition data:** Cần ≥1 lĩnh vực trong `DANH_MUC` + ≥1 đơn vị hợp lệ. Nếu filter lĩnh vực rỗng → precondition thiếu, note báo dev seed.
- Detail view có stepper 6 bước + 4 tab (Tiêu chí / Phân công / Thực hiện / Báo cáo) — ngoài scope Bước 2-3.

  |----------|--------|

## Bước 1: LOGIN

1. Mở `http://103.172.236.130:3000/`
2. `canbo_tw` / `Secret@123`
3. OTP `666666`
4. Đợi dashboard (< 5s)
5. Chụp `danhgia-login-dashboard.png`

| Kết quả | Action |
|---------|--------|
| ✅ Dashboard | → Bước 2 |
| ❌ Lock / sai pass / 500 | **STOP**, BLOCKED |

## Bước 2: VÀO MODULE

### 2a. Verify menu + (submenu nếu có) sidebar

1. `$B snapshot -i`
2. Grep `Đánh giá` hoặc `Đánh giá hiệu quả hỗ trợ pháp lý`
3. Nếu có submenu: click menu → đợi expand → grep `Kế hoạch Đánh giá`

| Kết quả | Action |
|---------|--------|
| ✅ Menu visible (+ submenu nếu có) | → 2b |
| ❌ Menu không thấy | Verify role. Thiếu → **FAIL**, báo FE |
| ❌ Submenu không expand | Retry 1 lần. Vẫn không → **FAIL** |

### 2b. Navigate

1. Click submenu đích (hoặc menu nếu không có submenu)
2. Đợi page render (< 5s)
3. Verify URL + filter trạng thái (≥7 state) + nút `+ Tạo đợt đánh giá`
4. Chụp `danhgia-page.png`

| Kết quả | Action |
|---------|--------|
| ✅ URL + elements | → Bước 3 |
| ❌ Redirect `/login` | Session lỗi, báo dev |
| ❌ Page trắng | **FAIL**, báo BE |
| ❌ Thiếu nhiều state trong filter | **FAIL**, note |

---

## Bước 3: KIỂM TRA LỖI NGẦM

1. `$B console --errors`
2. `$B network` → grep 4xx/5xx
3. `$B snapshot` → grep `"Lỗi"`, `"Validation failed"`, `"Không thành công"`, `"undefined"`

| Kết quả | Verdict |
|---------|---------|
| Sạch | ✅ PASS |
| Warning / 401 single | ⚠️ WARN |
| TypeError / 5xx / toast đỏ | ❌ FAIL |
