## Ghi chú đặc thù module
- **Account smoke:** **KHÔNG** dùng `canbo_tw` — CB NV không có quyền QTHT. Dùng `admin` hoặc `qtht_tw`.
- **Email activation link:** tài khoản mới tạo → state `CHO_KICH_HOAT`, gửi email link kích hoạt. Verify bằng MailHog (`http://103.172.236.130:8025`) nếu cần, ngoài scope smoke.
- **Khóa tự động:** đăng nhập sai password >5 lần → state `TAM_KHOA` auto-transition. Test API login trả `"tạm khóa"` → state đã đồng bộ.
- **Permission nhạy cảm:** module này tạo/xóa user — cần cực kỳ cẩn trọng với click trên bản ghi thật. Chỉ đọc list ở Bước 2-3, không click action Xóa.

## Pre-check (trước khi mở browse)

- [ ] Server up: `curl -o /dev/null -w "%{http_code}" http://103.172.236.130:3000/` → `200`
- [ ] Account admin/qtht_tw chưa lock:
  ```bash
  curl -s -X POST http://103.172.236.130:3000/api/v1/auth/login \
    -H "Content-Type: application/json" \
    -d '{"username":"qtht_tw","password":"Secret@123"}'
  # (hoặc admin / Secret@123)
  ```

  | Response | Action |
  |----------|--------|
  | `"otpToken"` | → Bước 1 |
  | `"tạm khóa"` | **STOP**, đổi sang account còn lại trong 2 account trên |
  | `"Invalid credentials"` | **STOP**, verify password — `admin` password là `Secret@123` (không phải Secret@123) |

## Bước 1: LOGIN

1. Mở `http://103.172.236.130:3000/`
2. Nhập account chọn + password tương ứng (xem Metadata)
3. OTP `666666`
4. Đợi dashboard (< 5s)
5. Chụp `taikhoan-login-dashboard.png`

| Kết quả | Action |
|---------|--------|
| ✅ Dashboard + sidebar có `Quản trị hệ thống` | → Bước 2 |
| ❌ Lock / sai pass / 500 | **STOP**, BLOCKED |

---

## Bước 2: VÀO MODULE

### 2a. Verify menu + submenu sidebar

1. `$B snapshot -i`
2. Grep `Quản trị hệ thống` — phải xuất hiện (có ▶ arrow)
3. Click menu `Quản trị hệ thống` → đợi submenu expand
4. Snapshot lại → grep `Tài khoản` (hoặc `Quản lý tài khoản`) — phải xuất hiện

| Kết quả | Action |
|---------|--------|
| ✅ Menu + submenu `Tài khoản` visible | → 2b |
| ❌ Không thấy menu QTHT | Account không có role QTHT — dùng `admin`/`qtht_tw` |
| ❌ Menu có, submenu `Tài khoản` không | Retry click 1 lần. Vẫn thiếu → **FAIL**, báo FE |

### 2b. Navigate

1. Click submenu `Tài khoản`
2. Đợi page render (< 5s)
3. Verify URL + tabs 4 state + nút `+ Tạo tài khoản` + header cột đầy đủ
4. Chụp `taikhoan-page.png`

| Kết quả | Action |
|---------|--------|
| ✅ URL + elements | → Bước 3 |
| ❌ Redirect `/login` / 403 | Role không đủ (dù đã dùng admin) → báo dev |
| ❌ Page trắng | **FAIL**, báo BE |
| ❌ Thiếu cột/tabs | **FAIL**, note |

## Bước 3: KIỂM TRA LỖI NGẦM

1. `$B console --errors`
2. `$B network` → grep 4xx/5xx (đặc biệt 403 vì module nhạy cảm permission)
3. `$B snapshot` → grep `"Lỗi"`, `"Không có quyền"`, `"Validation failed"`, `"undefined"`

| Kết quả | Verdict |
|---------|---------|
| Sạch | ✅ PASS |
| Warning / 401 single | ⚠️ WARN |
| TypeError / 5xx / 403 / toast đỏ | ❌ FAIL |
