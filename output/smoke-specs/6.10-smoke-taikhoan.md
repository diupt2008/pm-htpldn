## Ghi chú đặc thù module
- **Account smoke:** **KHÔNG** dùng `cb_nv_tw_01` — CB NV không có quyền QTHT. Dùng `qtht_01` hoặc `admin`.
- **Email activation link:** TK mới tạo → state `CHO_KICH_HOAT`, gửi email link kích hoạt. Token **vĩnh viễn nếu kích hoạt lần đầu** (TVV/CG/NHT theo FR-VIII-26 step 3), **30 phút nếu reset MK** đang HOAT_DONG. Verify qua MailHog `http://103.172.236.130:8025` nếu cần.
- **Khóa tự động:** đăng nhập sai password >=5 lần → state `TAM_KHOA` auto-transition (BR-AUTH-07). Auto unlock sau 30 phút HOẶC QTHT mở khóa thủ công.
- **CHO_PHAN_QUYEN state:** 🚫 BLOCKED chờ BA Q3 — sau FR-VIII-22 bypass, scenario nào còn trigger state này? Nếu seed test data state CHO_PHAN_QUYEN trống → KHÔNG log bug (chờ BA chốt). Xem [bug-report-r7-srs-fr10-inconsistency.md](../qa-reports/round7-2026-05-06/bug-reports/bug-report-r7-srs-fr10-inconsistency.md) BUG-SRS-FR10-003.
- **Password rule mới:** SRS v3.5 yêu cầu **ký tự đặc biệt** (FR-VIII-15 ERR-TK-03 line 721). Test tạo TK với password chỉ "Abcdef12" → phải lỗi ERR-TK-03 chứa từ "ký tự đặc biệt".
- **Permission nhạy cảm:** module này tạo/xóa user — cẩn trọng click trên bản ghi thật. Chỉ đọc list ở Bước 2-3, không click action Xóa.
- **DON_VI 2 tầng:** filter "Đơn vị" mở dropdown → cây phải có TW root + BN và ĐP cùng cấp 2 (không có ĐP nested dưới BN). Nếu thấy ĐP nested dưới BN → log bug FR-VIII-05.
- **BR-EC-05 đã bỏ:** rule "session limit 3 phiên / QTHT 1 phiên" KHÔNG có trong SRS v3.5 — không test.

1. Mở `http://103.172.236.130:3000/`
2. Nhập account chọn + password tương ứng (xem Metadata)
3. OTP `666666`

## Bước 2: VÀO MODULE

### 2a. Verify menu + submenu sidebar

1. `$B snapshot -i`
2. Grep `Quản trị hệ thống` — phải xuất hiện (có ▶ arrow)
3. Click menu `Quản trị hệ thống` → đợi submenu expand
4. Snapshot lại → grep `Tài khoản` (hoặc `Quản lý tài khoản`) — phải xuất hiện

| Kết quả | Action |
|---------|--------|
| ✅ Menu + submenu `Tài khoản` visible | → 2b |
| ❌ Không thấy menu QTHT | Account không có role QTHT — dùng `admin`/`qtht_tw` |
| ❌ Menu có, submenu `Tài khoản` không | Retry click 1 lần. Vẫn thiếu → **FAIL**, báo FE |

### 2b. Navigate

1. Click submenu `Tài khoản`
2. Đợi page render (< 5s)
3. Verify URL + **tabs 5 state** (gồm `CHO_PHAN_QUYEN` mới) + filter trạng thái 5 lựa chọn + nút `+ Tạo tài khoản` + header cột đầy đủ + cột Hành động có nút `Phân quyền` (cho row state=CHO_PHAN_QUYEN)
4. Chụp `taikhoan-page.png`

| Kết quả | Action |
|---------|--------|
| ✅ URL + 5 tabs + nút Phân quyền | → Bước 3 |
| ⚠️ Chỉ có 4 tabs (thiếu CHO_PHAN_QUYEN) | Log bug FE-FR10 — UI chưa apply SRS v3.5 (note BUG SRS unrelated) |
| ⚠️ Thiếu nút Phân quyền | Log bug FE-FR10 — fix C.5 chưa apply (SCR-VIII-03 row 16) |
| ❌ Redirect `/login` / 403 | Role không đủ (dù đã dùng admin) → báo dev |
| ❌ Page trắng | **FAIL**, báo BE |

## Bước 3: KIỂM TRA LỖI NGẦM

1. `$B console --errors`
2. `$B network` → grep 4xx/5xx (đặc biệt 403 vì module nhạy cảm permission)
3. `$B snapshot` → grep `"Lỗi"`, `"Không có quyền"`, `"Validation failed"`, `"undefined"`

| Kết quả | Verdict |
|---------|---------|
| Sạch | ✅ PASS |
| Warning / 401 single | ⚠️ WARN |
| TypeError / 5xx / 403 / toast đỏ | ❌ FAIL |
