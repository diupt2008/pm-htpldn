## Ghi chú đặc thù module

- **Test env `103.172.236.130:3000` hiện HTTP (không TLS):** Nếu không có endpoint HTTPS + mTLS → skip bước mTLS, chỉ smoke JWT. Mark trong report là `Smoke ở HTTP-only mode`.
- **JWT endpoint:** Xác nhận với dev có `/oauth/token` hoặc `/api/v1/auth/token` test-only hay không. Nếu không → xin sẵn 1 JWT tĩnh 24h từ dev team.
- **Data điều kiện:** Endpoint `/hoi-dap` chỉ trả bản ghi `DA_DUYET`. Nếu DB chưa có bản ghi DA_DUYET → response `data: []` là **PASS smoke** (API hoạt động), nhưng cần flag lại để Lệnh 3 tạo data trước khi chạy functional.

## Pre-check (trước khi bắn request)

- [ ] **Server up (health):**
  ```bash
  curl -o /dev/null -w "%{http_code}" http://103.172.236.130:3000/
  ```
  Kỳ vọng: `200`. Không thì **STOP**, báo DevOps.

- [ ] **JWT token có sẵn hoặc issue được:**
  - Option A — test env có `/oauth/token`:
    ```bash
    TOKEN=$(curl -s -X POST http://103.172.236.130:3000/api/v1/auth/token \
      -H "Content-Type: application/json" \
      -d '{"client_id":"cong-plqg-test","client_secret":"<secret>","scope":"htpldn:hoi-dap:read"}' \
      | jq -r .access_token)
    echo "$TOKEN" | head -c 40; echo "..."
    ```
  - Option B — xin JWT tĩnh từ dev, lưu env `TOKEN`.

  | Kết quả | Action |
  |---------|--------|
  | `TOKEN` dài ~200 ký tự (eyJ...) | → Bước 1 |
  | `null` / rỗng / 404 `/auth/token` | **STOP**, xin JWT tĩnh từ dev trước khi tiếp tục |
  | `401 invalid_client` | **STOP**, verify lại client_id/secret |

## Bước 1: VERIFY JWT VALID

Giải mã JWT (không verify signature, chỉ đọc claims) để chắc:

```bash
echo "$TOKEN" | cut -d. -f2 | base64 -d 2>/dev/null | jq .
```

Kỳ vọng payload có:
- `iss` = `htpldn.moj.gov.vn` (hoặc tương đương test env)
- `scope` chứa `htpldn:hoi-dap:read`
- `exp` > `now()`
- `consumer_id` (hoặc `sub`, `client_id`) có giá trị

| Kết quả | Action |
|---------|--------|
| ✅ Claims đầy đủ, exp còn hiệu lực | → Bước 2 |
| ❌ exp đã hết | Issue lại token hoặc xin token mới từ dev |
| ❌ scope thiếu `htpldn:hoi-dap:read` | Xin token có scope đúng — smoke sẽ fail ở bước 2 |
| ❌ JWT không parse được (không phải 3 phần) | **STOP**, báo dev auth server lỗi |

## Bước 2: GỌI ENDPOINT SMOKE

### 2a. Request cơ bản (không mTLS)

```bash
curl -i -s -X GET "http://103.172.236.130:3000/api/v1/hoi-dap?page=1&size=5" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Accept: application/json" \
  -o /tmp/api-smoke-body.json \
  -w "HTTP:%{http_code}\nTIME:%{time_total}\n" \
  | tee /tmp/api-smoke-meta.txt
cat /tmp/api-smoke-body.json | jq . | head -40
```

### 2b. Verify response envelope

Check các điều kiện:

```bash
# 1. HTTP status = 200
grep "HTTP:200" /tmp/api-smoke-meta.txt

# 2. Response time < 3s
awk -F: '/TIME:/ { if ($2 > 3) print "FAIL: >3s"; else print "PASS: "$2"s" }' /tmp/api-smoke-meta.txt

# 3. Envelope có đủ field
cat /tmp/api-smoke-body.json | jq '
  {
    has_success: (.success != null),
    success_value: .success,
    has_data: (.data != null and (.data | type) == "array"),
    has_pagination: (.pagination.page != null and .pagination.size != null and .pagination.total_elements != null),
    has_timestamp: (.timestamp != null),
    data_count: (.data | length)
  }
'
```

| Kết quả | Verdict |
|---------|---------|
| HTTP 200, envelope đủ 4 field (`success/data/pagination/timestamp`), time < 3s | ✅ PASS → Bước 3 |
| HTTP 200 nhưng envelope sai format (thiếu `pagination` hoặc `success` false khi đáng lẽ true) | ❌ FAIL — dev sai envelope format |
| HTTP 401 | ❌ FAIL — JWT không được chấp nhận, verify lại |
| HTTP 403 | ❌ FAIL — scope không đủ, request JWT với scope đúng |
| HTTP 404 | ❌ FAIL — endpoint không tồn tại ở env này |
| HTTP 429 | ⚠️ WARN — rate limit đã chạm (có ai đang load-test?), chờ 60s retry |
| HTTP 5xx | ❌ FAIL — BE lỗi, báo dev |
| Time > 3s | ⚠️ WARN — note nhưng không block smoke |

### 2c. Verify data filter publishable (BR-INTG-07)

Nếu `data_count > 0`, check mọi bản ghi đều có `trang_thai` thuộc publishable set:

```bash
cat /tmp/api-smoke-body.json | jq '
  .data
  | map(.trang_thai)
  | unique
  | map(select(. != "DA_DUYET" and . != "CONG_KHAI" and . != "HOAN_THANH"))
'
```

Kỳ vọng output: `[]` (không có bản ghi non-publishable nào lọt).

| Kết quả | Action |
|---------|--------|
| `[]` | ✅ Filter đúng — BR-INTG-07 PASS |
| `["MOI"]` hoặc các state khác | ❌ FAIL — leak bản ghi chưa duyệt, báo BE NGAY |

## Crash / Retry Rule (adapt từ template)

Module này KHÔNG dùng `/browse` nên không có crash kiểu Playwright. Thay vào đó:

| Lỗi | Action |
|-----|--------|
| `curl: (7) Failed to connect` | Server down → **STOP**, báo DevOps |
| `curl: (35) SSL` (chỉ nếu dùng HTTPS) | TLS issue → skip mTLS, fallback HTTP nếu env cho phép |
| JWT endpoint 404 lặp lại | Không tự retry, yêu cầu dev cấp JWT tĩnh |
| Endpoint `/hoi-dap` 5xx 2 lần liên tục | **STOP**, mark BLOCKED, báo BE |
