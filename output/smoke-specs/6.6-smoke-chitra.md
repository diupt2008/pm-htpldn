## Ghi chú đặc thù module
- **Precondition data:** HS chi trả thường được tạo từ DVC/LGSP (integration external) → state ban đầu `CHO_TIEP_NHAN`. Nội bộ test có thể cần dữ liệu seed sẵn.
- **Số tiền lớn:** UI có thể format theo VND (dấu phẩy / chấm). Verify không hiển thị `NaN` hoặc `undefined` trong cột số tiền.
- **Thanh toán external:** state `DA_THANH_TOAN` — integration với hệ thống kho bạc / ngân hàng, ngoài scope smoke.

## Bước 3: KIỂM TRA LỖI NGẦM

1. `$B console --errors`
2. `$B network` → grep 4xx/5xx
3. `$B snapshot` → grep `"Lỗi"`, `"Validation failed"`, `"Không thành công"`, `"undefined"`, `"NaN"`

| Kết quả | Verdict |
|---------|---------|
| Sạch | ✅ PASS |
| Warning / 401 | ⚠️ WARN |
| TypeError / 5xx / toast đỏ / NaN cột số tiền | ❌ FAIL |
