## Ghi chú đặc thù module
- **Guard chuyển state `VO_HIEU_HOA`:** có điều kiện (không còn VV / hỏi đáp đang xử lý). Ngoài scope smoke, chỉ note nếu UI báo lỗi.
- **State đổi tên:** `DANG_HOAT_DONG` (cũ) → **`HOAT_DONG`** (mới) đồng bộ enum CHECK constraint (cite `srs-fr-04:2011`).
- **State mới `CHO_KICH_HOAT`** sau phê duyệt — TVV/CG bấm link mail kích hoạt qua FR-VIII-26 → `HOAT_DONG`.
- **Smoke chỉ test SCR-IV-01** (Tư vấn viên / Chuyên gia). Smoke NHT (`SCR-IV-NHT-01`) + TC TV (`SCR-IV-NEW-01`) là sub-menu riêng — KHÔNG cover trong file này. Hiện NHT/TC TV có deploy gap (entity 404 / sub-menu chưa có) → defer.
- **NHT KHÔNG hiển thị** trong tab SCR-IV-01 vì đã tách entity riêng (`loai_tvv` enum chỉ `('TVV','CG')` — cite `srs-fr-04:1998`).

## Pre-check (trước khi mở browse)

- [ ] **Server up:**
  ```bash
  curl -o /dev/null -w "%{http_code}" http://103.172.236.130:3000/
  ```
  Kỳ vọng: `200`. Không thì **STOP**.

- [ ] **Account chưa lock:**
  ```bash
  curl -s -X POST http://103.172.236.130:3000/api/v1/auth/login \
    -H "Content-Type: application/json" \
    -d '{"username":"qtht_01","password":"Secret@123"}'
  ```

  | Response | Action |
  |----------|--------|
  | `"otpToken"` | → Bước 1 |
  | `"tạm khóa"` | **STOP**, đổi account |
  | `"Invalid credentials"` | **STOP**, verify password |

## Bước 2: VÀO MODULE

### 2a. Verify menu trong sidebar

1. `$B snapshot -i`
2. Grep `Quản lý chuyên gia / tư vấn viên`

| Kết quả | Action |
|---------|--------|

## Bước 3: KIỂM TRA LỖI NGẦM

1. `$B console --errors`
2. `$B network` → grep 4xx/5xx
3. `$B snapshot` → grep `"Lỗi"`, `"Validation failed"`, `"Không thành công"`, `"Có lỗi xảy ra"`, `"undefined"`

| Kết quả | Verdict |
|---------|---------|
| Sạch | ✅ PASS |
| Warning / 401 single | ⚠️ WARN |
| TypeError / 5xx / toast đỏ | ❌ FAIL |
