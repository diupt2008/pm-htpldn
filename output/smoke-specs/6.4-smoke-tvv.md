## Pre-check (trước khi mở browse)

- [ ] **Server up:**
  ```bash
  curl -o /dev/null -w "%{http_code}" http://103.172.236.130:3000/
  ```
  Kỳ vọng: `200`. Không thì **STOP**.

- [ ] **Account chưa lock:**
  ```bash
  curl -s -X POST http://103.172.236.130:3000/api/v1/auth/login \
    -H "Content-Type: application/json" \
    -d '{"username":"canbo_tw","password":"Secret@123"}'
  ```

  | Response | Action |
  |----------|--------|
  | `"otpToken"` | → Bước 1 |
  | `"tạm khóa"` | **STOP**, đổi account |
  | `"Invalid credentials"` | **STOP**, verify password |

## Bước 1: LOGIN

1. Mở `http://103.172.236.130:3000/`
2. Nhập `canbo_tw` / `Secret@123`
3. Submit → OTP `666666`
4. Đợi dashboard (< 5s)
5. Chụp `tvv-login-dashboard.png`

| Kết quả | Action |
|---------|--------|
| ✅ Dashboard + sidebar | → Bước 2 |
| ❌ "Tài khoản tạm khóa" | **STOP**, BLOCKED |

## Bước 2: VÀO MODULE

### 2a. Verify menu trong sidebar

1. `$B snapshot -i`
2. Grep `Quản lý chuyên gia / tư vấn viên`

| Kết quả | Action |
|---------|--------|
| ✅ Menu visible | → 2b |
| ❌ Không thấy | Verify role. Role đúng mà thiếu → **FAIL**, báo FE |

*(Không có submenu — click trực tiếp vào menu.)*

### 2b. Navigate

1. Click menu (hoặc goto URL)
2. Đợi page render (< 5s)
3. Verify URL + tabs 9 state + nút `+ Thêm mới`
4. Chụp `tvv-page.png`

| Kết quả | Action |
|---------|--------|
| ✅ URL + elements đủ | → Bước 3 |
| ❌ Redirect `/login` | Session lỗi, báo dev |
| ❌ Page trắng | **FAIL**, báo BE |
| ❌ Thiếu tabs/buttons | **FAIL**, note |

## Bước 3: KIỂM TRA LỖI NGẦM

1. `$B console --errors`
2. `$B network` → grep 4xx/5xx
3. `$B snapshot` → grep `"Lỗi"`, `"Validation failed"`, `"Không thành công"`, `"Có lỗi xảy ra"`, `"undefined"`

| Kết quả | Verdict |
|---------|---------|
| Sạch | ✅ PASS |
| Warning / 401 single | ⚠️ WARN |
| TypeError / 5xx / toast đỏ | ❌ FAIL |
