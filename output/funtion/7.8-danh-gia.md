## Bối cảnh

Module Theo dõi Đánh giá Hiệu quả Hỗ trợ Pháp lý tổ chức đợt đánh giá hiệu quả HTPLDN theo **TT17/2025/TT-BTP** với 2 tần suất: **Sơ bộ 6 tháng** + **Tròn năm** (không đột xuất — BR-LEGAL-08). Mỗi đợt có **2 vai trò đơn vị riêng biệt**: `don_vi_id` = cơ quan thực hiện đánh giá, `co_quan_duoc_danh_gia_id` = cơ quan được đánh giá (1:1, bắt buộc, có thể khác). Quy trình:

**Lập KH → Phân công → Chờ duyệt PC → Thực hiện chấm điểm → Báo cáo → Chờ phê duyệt BC → Hoàn thành** (+ nhánh hủy `* → HUY`)

Sau `HOAN_THANH`, CB NV thuộc `co_quan_duoc_danh_gia_id` được mở SCR-VI-01 ở chế độ read-only để xem kết quả (FR-VI-10 mới v3.5).

## State Machine — SM-DANHGIA (8 state v3.5 + HUY)

```
[*] → LAP_KE_HOACH
LAP_KE_HOACH → PHAN_CONG → CHO_DUYET_PC
                              ↓ (CB PD duyệt PC)
                           THUC_HIEN
                              ↓ (CB PD từ chối PC) → PHAN_CONG
THUC_HIEN → BAO_CAO (auto khi tất cả VV chấm xong)
BAO_CAO → CHO_PHE_DUYET → HOAN_THANH
                              ↓ (CB PD từ chối BC) → BAO_CAO
LAP_KE_HOACH/PHAN_CONG/THUC_HIEN/BAO_CAO → HUY (CB NV/PD hủy đợt)
```

**Bảng 8 trạng thái:**

| Trạng thái | Mô tả | Màu badge |
|-----------|-------|-----------|
| `LAP_KE_HOACH` | CB NV đang lập kế hoạch đợt đánh giá | Xám |
| `PHAN_CONG` | CB NV đang phân công người đánh giá | Xanh dương |
| `CHO_DUYET_PC` | CB NV trình, chờ CB PD duyệt phân công | Cam |
| `THUC_HIEN` | CB PD đã duyệt, đang chấm điểm vụ việc | Vàng |
| `BAO_CAO` | CB NV nhập điểm xong, đang lập báo cáo | Xanh dương đậm |
| `CHO_PHE_DUYET` | CB NV trình báo cáo, chờ CB PD duyệt | Cam đậm |
| `HOAN_THANH` | CB PD đã duyệt báo cáo, đóng đợt | Xanh lá |
| `HUY` | Đợt đánh giá bị hủy (transition từ 4 state chưa hoàn thành) | Đỏ |

**Rút gọn entity chính:**
- `KE_HOACH_DANH_GIA` (owned, rename từ `DOT_DANH_GIA`/`DANH_GIA_HQ`) — đợt đánh giá, mã `DG-YYYYMMDD-SEQ`. **16 fields v3.5** (gồm `co_quan_duoc_danh_gia_id` FK→DON_VI 1:1 + `file_dinh_kem` file[]).
- `TIEU_CHI_DANH_GIA` (owned, DM mở rộng) — tiêu chí, trọng số SUM = 100% (BR-CALC-04). `nhom_tieu_chi='HIEU_QUA_HTPL'`.
- `KET_QUA_DANH_GIA` (owned) — điểm chấm per VV. FK `ke_hoach_id` → KE_HOACH_DANH_GIA (rename từ `dot_danh_gia_id` v3.5).
- `BAO_CAO_DANH_GIA` (owned, 1:1 với KE_HOACH_DANH_GIA) — BC tổng hợp mẫu 21a/21b TT17/2025.
