## Bối cảnh

Module quản lý kho biểu mẫu/hợp đồng mẫu (HĐ lao động, HĐ chuyển nhượng, HĐ DV Marketing...) cho DNNVV tham khảo + tải về qua Cổng PLQG. Cấu trúc 2 cấp: **Thư mục → Biểu mẫu**. Công khai trực tiếp lên Cổng PLQG **KHÔNG cần phê duyệt** (BR-FLOW-07 — chỉ áp cho FR-VII-03 công khai thư mục).

**Quy trình:** Tạo Thư mục → Thêm Biểu mẫu (upload file + 4 trường công khai chuyên trang [CR-01]) → Công khai lên Cổng → DN tải về qua API.

## State Machine — SM-BIEUMAU

```
     ┌───────── CONG_KHAI ◄──────┐
     │              ▲            │
NHAP ┤              │            │
     │              ▼            │
     └───────────── AN ──────────┘
          │                 │
          ▼                 ▼
         XOA               XOA
```

- NHAP → CONG_KHAI: công khai thư mục (FR-VII-03 — không cần duyệt — BR-FLOW-07)
- CONG_KHAI ↔ AN: ẩn / công khai lại
- NHAP / AN → XOA: soft-delete (chỉ khi thư mục rỗng)

> **Phân biệt 2 cấp công khai (CR-01):**
> - **Cấp THU_MUC_BIEU_MAU** — `trang_thai` enum `NHAP/CONG_KHAI/AN` — thuộc FR-VII-03 (BR-FLOW-07).
> - **Cấp BIEU_MAU** — Switch `cong_khai` boolean + 4 trường công khai chuyên trang — thuộc FR-VII-04 (BR-PUBLIC-01/02/03).

## Ghi chú thực thi

- **Authorization (BM-032 → BM-036):** Smoke-level. Full coverage ở [permission-matrix.md](../permission-matrix.md).
- **API Cổng PLQG (BM-038):** Dùng Postman/Bruno gọi trực tiếp endpoint. Verify chỉ trả về record `cong_khai`=1, filter hoạt động.
- **File test (BM-004, BM-015, BM-016):** Chuẩn bị sẵn:
  - `sample-valid.docx` (≤20MB) — happy path
  - `sample-oversize.docx` (>20MB) — test limit
  - `sample-invalid.txt` / `sample.exe` — test format
  - `sample-macro.docm` — test virus scan (EC-02)
- **Preview (BM-007):** Verify cả doc/docx (convert → PDF) và xls/xlsx (bảng read-only). Nếu không hỗ trợ preview → fallback "Tải về".
- **Import mixed (BM-029):** Upload 5 file = 3 hợp lệ + 1 >20MB + 1 sai format → verify 3 thành công, 2 lỗi có chi tiết.
- **HĐ Tư vấn:** Đã chuyển sang [7.14-hop-dong-tv.md](7.14-hop-dong-tv.md) per SRS v3.5 (FR-14). 3 TC cũ BM-030/031/037 strike-through trong matrix làm dấu hiệu di chuyển.
- **Audit (BM-040):** Verify trên màn Nhật ký HT — action codes: `CREATE_BIEU_MAU`, `UPDATE_BIEU_MAU`, `DELETE_BIEU_MAU`, `PUBLISH`, `UNPUBLISH`, `TAI_BIEU_MAU`, `BULK_IMPORT`.
- **Công khai thư mục (BM-022):** Verify cả trạng thái DB (`trang_thai`=CONG_KHAI) và record thực sự appear trên Cổng PLQG (qua API call BM-038).
- **Switch công khai BIEU_MAU (BM-041 → BM-046) [CR-01]:** Test trên SCR-VII-02 form Thêm/Sửa biểu mẫu. Switch OFF mặc định khi tạo mới. Bật → 3 trường (ảnh đại diện / mô tả công khai / file đính kèm công khai) hiện ngay; tắt → ẩn + clear value khi save.
- **Auto-fill `thoi_gian_dang_tai` (BM-042, BM-044) [CR-01]:** Verify DB column dạng datetime, set NOW() ngay tại API endpoint update — không phải client-side. Field read-only trên form, không có input element editable.
- **Reject công khai bản ghi Hủy (BM-045) [CR-01]:** Nếu workflow trạng thái BIEU_MAU mở rộng có Hủy/Từ chối → test bật Switch ở các state đó → backend reject. Ghi observation nếu BE chưa enforce.
- **Cột "Đã công khai" + "Ảnh đại diện" (BM-046, BM-047) [CR-01]:** Verify SCR-VII-02 render đúng cột mới sau CR-01. Tooltip hover badge xanh phải show `thoi_gian_dang_tai` định dạng `YYYY-MM-DD HH:mm`.
