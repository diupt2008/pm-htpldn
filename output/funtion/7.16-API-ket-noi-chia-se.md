## Bối cảnh

Module API Kết nối Chia sẻ Dữ liệu cung cấp **18 API outbound** (9 cặp: chia sẻ + tìm kiếm) cho hệ thống khác (chủ yếu **Cổng PLQG**) qua REST JSON. Kết nối **trực tiếp (không qua LGSP)**, bảo mật 2 lớp **mTLS + JWT Bearer RS256**, rate limit **100 req/min/consumer**, response time **< 3s**.

**Đặc thù:**
- **Không có màn hình CMS** — chỉ API outbound. Giám sát qua Dashboard MH-01, Grafana, AUDIT_LOG.
- **Read-only:** Không có state machine, không thay đổi dữ liệu PM.
- **Data filter:** CHỈ trả bản ghi ở trạng thái publishable (`DA_DUYET` / `CONG_KHAI` / `HOAN_THANH` / `DA_CONG_BO` / `DA_DUYET_BC` / `HOAT_DONG`) — theo BR-INTG-07.

**Cách tạo:**
- Data entity → walk workflow theo §6 của module nguồn (reuse data từ Lệnh 3 các module 7.2→7.9, 7.12).

**Ghi chú test env:** Nếu dev env KHÔNG có mTLS (simplified auth) → skip API-005, note BLOCKED trong report, vẫn chạy TC JWT.

## Ghi chú thực thi

**Công cụ test (KHÔNG dùng `/browse`):**
- **Postman / Bruno / Insomnia** cho chain request + assertion trên envelope.
- **curl + jq** cho ad-hoc verify + CI script.
- **k6 / Artillery** cho rate-limit (API-007, API-034) + response time p95 (API-010).
- **OpenSSL `s_client`** test mTLS handshake (API-005).

**Template curl cơ bản:**
```bash
# Obtain JWT (giả định endpoint test /oauth/token)
TOKEN=$(curl -s -X POST https://htpldn-test.moj.gov.vn/oauth/token \
  -d "grant_type=client_credentials&client_id=cong-plqg-test&scope=htpldn:hoi-dap:read" \
  --cert client.crt --key client.key | jq -r .access_token)

# Gọi API
curl -i https://htpldn-test.moj.gov.vn/api/v1/hoi-dap?page=1&size=20 \
  -H "Authorization: Bearer $TOKEN" \
  --cert client.crt --key client.key
```

**Cảnh báo môi trường:**
- Test env `103.172.236.130:3000` hiện dùng **HTTP (không TLS)** + bypass OTP 666666. Dev cần xác nhận có test-only JWT endpoint và chấp nhận skip mTLS trên env này.
- Nếu skip mTLS → mark API-005 `BLOCKED — test env không có TLS handshake`, retest ở staging có cert.

**Auth (API-041, API-042):** Smoke-level cho DN role. Full coverage ở [permission-matrix.md](../permission-matrix.md) row DN.

**Cross-module dependencies:**
- API-012/017/019/025/027 yêu cầu data entity ở cả state publishable + non-publishable → chạy SAU Lệnh 3 của module 7.2/7.4/7.5/7.12.
- API-031/032 yêu cầu approve/publish workflow hoạt động → chạy SAU module 7.2/7.9 đã PASS functional.

**AUDIT_LOG verify (API-035):**
- Sau khi chạy 1 request, query DB: `SELECT * FROM AUDIT_LOG WHERE endpoint LIKE '/api/v1/%' ORDER BY timestamp DESC LIMIT 1` → verify có row với đúng consumer_id.

**Rate-limit test (API-007):**
- Dùng script bash `for i in {1..101}; do curl ... & done; wait` hoặc k6 với 101 VUs constant-arrival-rate.
- Reset rate-limit counter giữa các TC bằng cách chờ 60s hoặc dùng 2 consumer khác nhau.
