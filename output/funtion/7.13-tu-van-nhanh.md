## Bối cảnh

Module Tư vấn Nhanh là hệ thống tra cứu Q&A pháp lý (keyword search, KHÔNG AI) phục vụ DN tự tra cứu trên Cổng PLQG + cho phép CB NV gợi ý trả lời từ kho dựng sẵn với quy trình:
**DN gửi câu hỏi → Keyword search kho → Gợi ý TOP 5 → CB NV chọn/chỉnh sửa → Gửi trả lời → DN đánh giá**

**Kho câu hỏi có 3 nguồn bổ sung (BR-FLOW-10):**
1. `TU_DONG` — HOI_DAP (Nhóm II) DA_DUYET → tự động vào kho, KHÔNG cần duyệt lại
2. `THU_CONG` — CB NV nhập → CHO_DUYET → CB PD duyệt
3. `IMPORT` — Upload Excel → CHO_DUYET → CB PD duyệt

## State Machine — SM-TVNHANH

```
MOI → DANG_TIM_KIEM → DA_GOI_Y → CB_TRA_LOI → HOAN_THANH
                         ↓
                      HOAN_THANH (DN hài lòng, đánh giá trực tiếp)
 ↓
HET_HAN (auto 30 ngày không xử lý)
```

**Trạng thái:**

| Trạng thái | Mô tả |
|-----------|-------|
| MOI | DN vừa gửi câu hỏi qua Cổng PLQG |
| DANG_TIM_KIEM | HT đang keyword search kho Q&A |
| DA_GOI_Y | HT đã gợi ý TOP 5 câu trả lời |
| CB_TRA_LOI | CB NV đang/đã soạn + gửi trả lời |
| HOAN_THANH | DN đã đánh giá, kết thúc phiên |
| HET_HAN | Quá 30 ngày không xử lý (batch job) |

**Cách tạo data:** Walk workflow theo SM-TVNHANH.
- **Kho Q&A (thủ công):** `canbo_tw` tạo Q&A → `lanhdao_tw` duyệt → DA_DUYET.
- **Kho Q&A (tự động):** Seed HOI_DAP DA_DUYET bằng module 7.2 → trigger auto-import.
- **Phiên TV nhanh:** Dùng `dn_user` gửi câu hỏi qua API Cổng PLQG (hoặc stub DB trực tiếp nếu chưa có Cổng). `canbo_tw` trả lời → `lanhdao_tw` không cần (TV nhanh không có phê duyệt bản ghi phiên).
- **HET_HAN:** Set `cau_hinh.tvnhanh_timeout = 1 phút` để test batch, hoặc stub `created_at` lùi 31 ngày.

- **Auto-timeout SM-TVNHANH:** MOI > 30 ngày → HET_HAN + TB CB NV (TVN-032)

## Ghi chú thực thi

- **Authorization (TVN-033 → TVN-036):** Smoke-level. Full coverage ở [permission-matrix.md](../permission-matrix.md). DN role không có CMS access — UC160/UC161/UC162 chạy qua API Cổng PLQG (Postman/Bruno), không `/browse`.
- **Auto-import (TVN-014):** Setup trigger DB hoặc event handler. Verify 2 điều kiện:
  - HOI_DAP chuyển sang DA_DUYET lần đầu → tạo 1 KHO_CAU_HOI mới, state = DA_DUYET, nguồn = TU_DONG
  - HOI_DAP chuyển DA_DUYET lần 2 (sau revoke + re-duyệt) → KHÔNG tạo bản ghi trùng (idempotent theo `hoi_dap_goc_id`)
- **Full-text search (TVN-002, TVN-015, TVN-017, TVN-026):** Verify GIN index có trên tsvector(`cau_hoi + cau_tra_loi + tu_khoa`). Test unaccent (tìm "hop dong" ra "hợp đồng"). Nếu BE chưa có GIN index → ghi nhận BLOCKED.
- **Notification (TVN-010, TVN-011, TVN-019, TVN-022, TVN-032):** Verify email/in-app qua MailHog — xem [§9.1](../test-strategy.md#9-môi-trường-kiểm-thử). TVN-019 gửi TB cho DN qua Cổng PLQG (webhook outbound — stub nếu chưa có hạ tầng).
- **API inbound UC160/UC161/UC162:** Dùng Postman/Bruno gọi `POST /api/v1/cong-plqg/tv-nhanh/hoi`, `GET /api/v1/cong-plqg/tv-nhanh/search`, `POST /api/v1/cong-plqg/tv-nhanh/danh-gia` với header `X-API-Key`. Verify:
  - 200 OK + body đúng schema outputs §SRS
  - DB có bản ghi mới với `nguon = CONG_PLQG` (đối với phiên TV)
  - Upsert DN theo MST nếu thiếu
- **Auto-timeout (TVN-032):** Set `cau_hinh.tvnhanh_timeout_ngay = 1/1440` (≈1 phút) trên môi trường test hoặc gọi endpoint batch job manual để test nhanh. Nếu không có quyền sửa config → ghi nhận BLOCKED, dev hỗ trợ.
- **Rich Text Editor (TVN-003, TVN-018):** Dùng `$B type` trên `[contenteditable="true"]` hoặc `.ql-editor`. Verify output HTML được escape khi hiển thị cho DN qua Cổng (XSS smoke test).
- **Scoped permission (TVN-034):** Test với `canbo_bn` (Bộ KH&ĐT) vs `canbo_tinh` (Sở TP HN) vs `canbo_tw` (TW) — verify data isolation ngang cấp + visibility kho nguồn TU_DONG.
- **Import Excel (TVN-005, TVN-006):** Chuẩn bị sẵn file test:
  - `kho-qa-valid.xlsx` — 10 dòng hợp lệ (cột: cau_hoi, cau_tra_loi, linh_vuc, tu_khoa)
  - `kho-qa-invalid-format.csv` — file .csv thay vì .xlsx
  - `kho-qa-missing-column.xlsx` — thiếu cột bắt buộc
- **Cross-module dependency:**
  - TVN-014 / TVN-037 cần ≥1 HOI_DAP DA_DUYET → seed bằng module 7.2 (Hỏi đáp) trước.
  - TVN-034 cần ≥2 đơn vị (BN + TW) có data riêng → seed bằng account `canbo_bn` + `canbo_tw`.
