# 7.1 Module: Dashboard (11 FR)

> **Nguồn:** Tách từ [test-strategy.md §7.1](../test-strategy.md#71-module-dashboard-11-fr)

## Bối cảnh

Module Dashboard là **trang chủ** sau đăng nhập — hiển thị 9 chỉ số KPI + 2 biểu đồ + 2 KPI bổ sung. Module **read-only**, KHÔNG có CUD, chỉ đọc/tổng hợp dữ liệu từ các entity workflow của nhóm II/III/IV/V/VI/VII:

**Luồng dữ liệu tổng quan:**

```
Đăng nhập → Dashboard → 7 thẻ KPI (UC1-UC7) + 2 biểu đồ (UC8-UC9) + 2 KPI bổ sung (KPI-03, KPI-04)
                        │
                        ├── Click thẻ KPI → drill-down tới module chi tiết (II/III/IV/V)
                        └── Auto-refresh 60s (FR-I-CROSS-02)
```

**Đặc thù module:**
- Read-only — không thay đổi dữ liệu
- Scoped theo đơn vị (BR-AUTH-08): TW toàn quốc, BN/ĐP scoped
- Auto-refresh 60 giây (pause khi tab ẩn — Page Visibility API)
- Bộ lọc: `tu_ngay` / `den_ngay` / `nam` / `don_vi_id` → đồng bộ URL
- Drill-down: click thẻ KPI → điều hướng danh sách chi tiết (trừ 2 KPI bổ sung)

**Tham chiếu chéo:**
- [§1.2 Tài khoản test](../test-strategy.md#12-tài-khoản-test) — `canbo_tw`, `canbo_bn`, `canbo_tinh`, `lanhdao_*`, `qtht_tw`
- [§6 State Machine](../test-strategy.md#6-test-theo-state-machine) — Dashboard KHÔNG có SM riêng, đọc data của SM-HOIDAP / SM-VUVIEC / SM-KHOAHOC / SM-TVV
- [permission-matrix.md](../permission-matrix.md) — phân quyền xem Dashboard

## Data Readiness cho module này

> Dashboard là read-only. Data được chuẩn bị sẵn ở các module nguồn — không cần tạo riêng cho Dashboard, NHƯNG phải có ≥1 bản ghi ở mỗi state cần đếm, nếu không KPI về 0 và TC cross-module không verify được.

| # | Dữ liệu cần | Nguồn module | Test Path |
|---|-------------|--------------|-----------|
| 1 | ≥1 HOI_DAP `MOI` trong kỳ lọc | 7.2 Hỏi đáp | DASH-028 |
| 2 | ≥1 VU_VIEC mỗi trạng thái (tiếp nhận / đang xử lý / hoàn thành) | 7.5 Vụ việc | DASH-029 |
| 3 | ≥1 KHOA_HOC `DANG_DIEN_RA` + ≥1 `KET_THUC` trong kỳ | 7.3 Đào tạo | DASH-030 |
| 4 | ≥1 TU_VAN_VIEN `DANG_HOAT_DONG` | 7.4 CG/TVV | DASH-031 |
| 5 | ≥1 KET_QUA_DANH_GIA có `diem_tong` | 7.8 Đánh giá | DASH-032 |
| 6 | ≥1 KET_QUA_DAO_TAO có `xep_loai`, `diem_kiem_tra` | 7.3 Đào tạo | DASH-033 |
| 7 | ≥1 VU_VIEC `HOAN_THANH` đủ field `ngay_tiep_nhan` + `ngay_hoan_thanh` | 7.5 Vụ việc | DASH-034 |
| 8 | Data ở **nhiều đơn vị** (TW + ≥1 BN + ≥1 ĐP) | 7.5/7.2/7.3 | DASH-021/022/023 |

**Cách kiểm tra nhanh (smoke Lệnh 2):** Login `canbo_tw` → Dashboard → nếu mọi KPI = 0 → Data Readiness FAIL → chạy lại Lệnh 3 các module nguồn (7.2, 7.3, 7.4, 7.5, 7.8) trước. KHÔNG bypass bằng cách skip TC cross-module.

## Business Rules áp dụng

- **BR-AUTH-04:** Cấp cha thấy cấp con — TW thấy toàn quốc (DASH-021)
- **BR-AUTH-08:** Phân quyền dữ liệu theo `don_vi_id` — áp dụng cho mọi KPI (DASH-021..023)
- **BR-SLA-05:** Tỷ lệ tuân thủ SLA = COUNT(hoàn thành đúng hạn) / COUNT(hoàn thành) × 100% (DASH-032)

## Ghi chú thực thi

- **Drill-down URLs (DASH-016..019):** Verify cả (a) URL đổi đúng path + query, (b) module đích load đúng filter (tab trạng thái được active + count khớp KPI vừa click). Nếu chênh → bug cross-module priority cao.
- **Auto-refresh (DASH-013):** Dùng `$B network` ghi timestamp → chờ ≥60s → ghi lại → verify có batch request KPI mới. Không đo bằng mắt.
- **Page Visibility (DASH-014/015):** Dùng `$B js "document.hidden"` + mở tab khác/quay lại. Thay vì sleep 60s, verify rằng khi `document.hidden=true` thì KHÔNG có request trong network log.
- **Cross-module count (DASH-028..031):** So sánh số trên Dashboard vs count trên màn list module đích (cùng filter kỳ + đơn vị). Chênh >0 = bug — thường do (a) scope đơn vị khác nhau, (b) soft-delete không lọc đồng bộ, (c) timezone bộ lọc khác nhau.
- **BR-AUTH-03 ngang cấp (DASH-022):** Bắt buộc có data ở ≥2 BN khác nhau trong DB để thực sự verify — nếu chỉ 1 BN thì test inconclusive. Check Data Readiness #8.
- **BR-CALC-03 ngày làm việc (DASH-034):** Chuẩn bị mẫu VV có `ngay_tiep_nhan=Thứ 6` + `ngay_hoan_thanh=Thứ 2 tuần sau` → AVG phải là **1 ngày LV** (không phải 3 ngày). Nếu hệ thống tính 3 ngày → BR-CALC-03 sai.
- **Authorization (DASH-021..027):** Smoke-level. Full coverage ở [permission-matrix.md](../permission-matrix.md) — mỗi role × entity nguồn (HOI_DAP, VU_VIEC, KHOA_HOC, TU_VAN_VIEN).
- **Không có state machine:** Module read-only → Lệnh 3 "tạo data" cho Dashboard = chạy Lệnh 3 cho các module nguồn (7.2/7.3/7.4/7.5/7.8). KHÔNG cần data-readiness riêng cho 7.1 ngoài bảng trên.
- **Filter + URL sync (SCR-I-01):** Verify URL query params (`?tu_ngay=...&den_ngay=...&don_vi_id=...`) khớp với giá trị filter đang chọn — để bookmark/share URL hoạt động.
