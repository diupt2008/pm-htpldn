## Bối cảnh

Module Tư vấn Chuyên sâu xử lý vòng đời yêu cầu tư vấn 1-1 giữa DN ↔ Chuyên gia/TVV với quy trình:
**Tiếp nhận → Phân công CG → CG xác nhận → Đang tư vấn → Hoàn thành → Phê duyệt → Đã duyệt (gửi DN)**

Module có 4 UC loại B (CMS browser) + 3 UC loại M (API inbound từ Cổng PLQG):
- **CMS:** Quản lý nội dung TVCS (UC147), Tìm kiếm (UC148), Quản lý hồ sơ pháp lý DN (UC150), Quản lý tư liệu pháp lý vụ việc (UC152)
- **API inbound:** Tiếp nhận TVCS (UC149), Tiếp nhận HSPL (UC151), Tiếp nhận đánh giá chất lượng (UC153)

## State Machine — SM-TVCS

```
TIEP_NHAN → PHAN_CONG → DANG_TU_VAN → HOAN_THANH → CHO_PHE_DUYET → DA_DUYET
   ↓           ↓ ↑           ↓                          ↓
  HUY        HUY (CG từ chối)  HUY (DN+CB PD duyệt)  DANG_TU_VAN (từ chối)
              ↑
        TIEP_NHAN (timeout 2 ngày LV → auto-reject)
```

**Trạng thái:**

| Trạng thái | Mô tả |
|-----------|-------|
| TIEP_NHAN | YC TV mới tạo / tiếp nhận từ Cổng |
| PHAN_CONG | Đã phân công CG/TVV, chờ xác nhận |
| DANG_TU_VAN | CG đang tư vấn (sau xác nhận) |
| HOAN_THANH | CG hoàn thành (kèm VB TVPL) |
| CHO_PHE_DUYET | Auto-transition, chờ CB PD duyệt |
| DA_DUYET | Đã duyệt → gửi KQ cho DN, mở đánh giá |
| HUY | Hủy yêu cầu |

- **BR-DATA-05:** Audit trail mọi CUD + chuyển trạng thái + phê duyệt (TV-038)

- **BR-DATA-08:** Full-text search tiếng Việt unaccent (TV-005)
- **BR-FLOW-01:** Auto-transition HOAN_THANH → CHO_PHE_DUYET (TV-013)
- **BR-FLOW-04:** Từ chối phê duyệt bắt buộc lý do ≥10 ký tự (TV-015)
- **BR-FLOW-07:** Tư liệu PL công khai trực tiếp Cổng PLQG, KHÔNG cần duyệt (TV-024)
- **BR-NOTIF-01:** Auto thông báo khi có data mới từ API inbound Cổng PLQG (TV-027, TV-028, TV-029)

- **SLA phân công CG:** CG xác nhận/từ chối trong 2 ngày LV → quá hạn auto-reject về TIEP_NHAN (TV-011)

## Ghi chú thực thi

- **Authorization (TV-037 → TV-039):** Smoke-level. Full coverage ở [permission-matrix.md](../permission-matrix.md). Lưu ý DN role = 🔌 (API only), không có CMS access cho module này.
- **Auto-transition (TV-013):** Verify cả UI (badge HOAN_THANH → CHO_PHE_DUYET) và DB (state thực sự chuyển + có TB CB PD trong queue).
- **Notification (TV-007, TV-009, TV-014, TV-027):** Verify email/in-app qua MailHog — xem [§9.1](../test-strategy.md#9-môi-trường-kiểm-thử). Lưu ý: TV-027 (API inbound) phải gửi cho CB NV phụ trách theo đơn vị (theo upsert DN).
- **API inbound (TV-027, TV-028, TV-029):** Dùng Postman/Bruno gọi `POST /api/v1/cong-plqg/tvcs|hspl|danh-gia` với header `X-API-Key: <key>` và `Content-Type: application/json`. Verify:
  - HTTP 200 + body `{ket_qua: THANH_CONG, ma_noi_dung: TVCS-..., ngay_tiep_nhan: ISO}`
  - DB có bản ghi mới với `nguon = CONG_PLQG`
  - DOANH_NGHIEP upsert đúng theo MST
- **Idempotency (TV-029):** Gửi cùng payload với `hanh_dong=GUI_LAI` 2-3 lần → chỉ tạo 1 record, không ghi đè.
- **Scoped permission (TV-038):** Test với `cb_nv_bn` (chỉ thấy data BN mình) vs `cb_nv_tw` (thấy tất cả).
- **Cross-module dependency:**
  - TV-007/TV-031 cần ≥1 TVV `DANG_HOAT_DONG` → seed bằng module 7.3 (CG/TVV) trước.
  - TV-041 cần ≥1 VU_VIEC → seed bằng module 7.4.
  - TV-042 cần MST chưa tồn tại trong DB → verify auto-create DN.
- **File test:** Chuẩn bị sẵn `sample-valid.pdf` (<20MB), `sample-oversize.pdf` (>20MB), `sample-macro.docm` (test virus scan EC-FILE-01).
- **SLA timeout (TV-011):** Set `cau_hinh_sla.tvcs_xacnhan_cg = 1 phút` để test nhanh trên môi trường test (mặc định 2 ngày LV). Nếu không có quyền sửa config → ghi nhận BLOCKED, dev hỗ trợ.
