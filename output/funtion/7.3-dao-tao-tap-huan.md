## Bối cảnh

Module Đào tạo, Tập huấn quản lý THÔNG TIN chương trình đào tạo/tập huấn bồi dưỡng kiến thức pháp luật cho DNNVV (KHÔNG phải LMS đầy đủ — Out-of-Scope OS-04). Cấu trúc 2 cấp:

**Chương trình đào tạo (CTDT) → Khóa học → (Bài giảng N-N / Học viên / Điểm danh / Kết quả)**

Quy trình khóa học: **Dự thảo → Trình duyệt → Duyệt → Diễn ra → Kết thúc → Trình duyệt KQ → Hoàn thành → Chứng nhận**

2 hình thức tương đương: Trực tuyến và Trực tiếp. Luồng phê duyệt CB NV cùng cấp tạo → CB PD cùng cấp duyệt (BR-FLOW-03).

## State Machine — SM-KHOAHOC

```
DU_THAO → CHO_DUYET → DA_DUYET → DA_CONG_KHAI → DANG_DIEN_RA → DA_KET_THUC → CHO_DUYET_KQ → HOAN_THANH
            ↓             ↓                                                         ↓
         DU_THAO         HUY                                                    DA_KET_THUC
       (từ chối)      (chưa có ĐK)                                             (từ chối KQ)
  ↓
 HUY
```

**Auto-transition:**
- AT-01: CB NV nhấn "Gửi phê duyệt" → DU_THAO → CHO_DUYET
- AT-02: CB NV nhấn "Trình duyệt KQ" → DA_KET_THUC → CHO_DUYET_KQ

## Ghi chú thực thi

- **Authorization (DT-032 → DT-036):** Smoke-level. Full coverage ở [permission-matrix.md §8.1](../permission-matrix.md).
- **Auto-transition (DT-020, DT-024):** Verify cả UI (nút "Gửi phê duyệt" / "Trình duyệt KQ") và DB (trạng thái thực sự chuyển).
- **Notification (DT-028 hủy khóa học):** Verify email/notification qua MailHog — xem [§9.1](../test-strategy.md#9-môi-trường-kiểm-thử).
- **Scoped permission (DT-034):** Test với `lanhdao_tw` (toàn bộ) và `lanhdao_bn` / `lanhdao_dp` (chỉ đơn vị mình).
- **API Cổng PLQG (DT-027):** Nếu môi trường test không có API thực → stub/mock; ghi BLOCKED + lý do vào report.
- **Chứng nhận PDF (DT-031):** Verify file PDF sinh đúng template + đầy đủ thông tin HV/khóa/ngày cấp/số CN.
- **Import Excel kết quả (DT-011):** Test cả happy (đúng template) và error (ERR-KQ-02 sai format, ERR-KQ-03 mã HV không tồn tại) — lock hàng EC-03 khi 2 CB NV import đồng thời.
- **GV Portal (DT-010):** Account `giangvien_user` dùng để verify mặt chuyên trang GV (nếu có).
