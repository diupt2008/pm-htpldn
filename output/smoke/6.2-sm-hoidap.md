## Test Paths

```
Test Paths:
├── TP-HD-01: Happy Path
│   MOI → TIEP_NHAN → DANG_XU_LY → DA_TRA_LOI → CHO_PHE_DUYET → DA_DUYET → CONG_KHAI → HOAN_THANH
│
├── TP-HD-02: Reject & Return
│   ... → CHO_PHE_DUYET → DANG_XU_LY (reject, yêu cầu lý do BR-FLOW-04)
│   → sửa → DA_TRA_LOI → CHO_PHE_DUYET → DA_DUYET
│
├── TP-HD-03: Cancel
│   MOI → HUY
│
├── TP-HD-04: Auto-transition (BR-FLOW-01)
│   CB NV tích "Đã trả lời" → Tự động chuyển CHO_PHE_DUYET
│
├── TP-HD-05: Batch Approve (BR-FLOW-02)
│   CB PD chọn nhiều bản ghi → Phê duyệt hàng loạt
│
├── TP-HD-06: Immutability (BR-FLOW-03)
│   DA_DUYET → Cố sửa → Phải bị chặn
│
└── TP-HD-07: Unpublish
    CONG_KHAI → DA_DUYET (thu hồi công khai)
```

## Business Rules áp dụng

- **BR-FLOW-01:** Auto-transition khi tick "Đã trả lời" → CHO_PHE_DUYET
- **BR-FLOW-02:** Batch approve — CB PD duyệt nhiều bản ghi cùng lúc
- **BR-FLOW-03:** Immutability — không sửa/xóa sau DA_DUYET
- **BR-FLOW-04:** Từ chối bắt buộc nhập lý do
- **BR-SLA-01:** Deadline tính từ ngày tiếp nhận + N ngày LV
- **BR-SLA-02:** 4 mức cảnh báo SLA

## Data Readiness

Để chạy đủ 7 test paths, cần dữ liệu ở các state:

| State | Mục đích test |
|-------|---------|
| MOI | TP-HD-01, TP-HD-03 |
| TIEP_NHAN | TP-HD-01 |
| DANG_XU_LY | TP-HD-01, TP-HD-02 (sau reject) |
| DA_TRA_LOI | TP-HD-04 |
| CHO_PHE_DUYET | TP-HD-02, TP-HD-05 |
| DA_DUYET | TP-HD-06, TP-HD-07 |
| CONG_KHAI | TP-HD-07 |

## Ghi chú thực thi

- **TP-HD-04 (auto-transition):** Verify cả UI (toggle) và DB (state thực chuyển).
- **TP-HD-05 (batch approve):** Tạo ≥5 bản ghi ở CHO_PHE_DUYET, duyệt 1 lần, verify tất cả → DA_DUYET.
- **TP-HD-06 (immutability):** Sau DA_DUYET, thử sửa qua UI và API — cả 2 đều phải bị chặn.
