## Danh sách trạng thái

| State | Ý nghĩa |
|-------|---------|
| CHO_KICH_HOAT | Mới tạo, chờ kích hoạt (email link) |
| HOAT_DONG | Active — đăng nhập được |
| TAM_KHOA | Bị khóa tạm thời (auto/manual) |
| VO_HIEU_HOA | Đã vô hiệu hóa (có thể do hết hạn kích hoạt) |

## Test Paths

```
Test Paths:
├── TP-TK-01: Happy Path
│   CHO_KICH_HOAT → HOAT_DONG
│
├── TP-TK-02: Khóa tự động
│   HOAT_DONG → TAM_KHOA (sau 5 lần sai mật khẩu — BR-AUTH-07)
│
├── TP-TK-03: Mở khóa tự động
│   TAM_KHOA → HOAT_DONG (sau 30 phút)
│
├── TP-TK-04: QTHT mở khóa thủ công
│   TAM_KHOA → HOAT_DONG (QTHT unlock)
│
├── TP-TK-05: Vô hiệu hóa
│   HOAT_DONG → VO_HIEU_HOA
│
├── TP-TK-06: Hết hạn kích hoạt
│   CHO_KICH_HOAT → VO_HIEU_HOA (auto sau 7 ngày)
│
└── TP-TK-07: Session limit (BR-EC-05)
    Login 3 phiên → phiên thứ 4 hủy phiên cũ nhất; QTHT giới hạn 1 phiên
```

## Business Rules áp dụng

- **BR-AUTH-06:** Session timeout 30 phút idle
- **BR-AUTH-07:** Khóa tài khoản sau 5 lần sai mật khẩu (TP-TK-02)
- **BR-EC-05:** Session limit 3 phiên/user, QTHT 1 phiên (TP-TK-07)
- **BR-SEC-06:** Mật khẩu ≥8 ký tự, hoa+thường+số

## Data Readiness

| State | Mục đích test |
|-------|---------|
| CHO_KICH_HOAT | TP-TK-01, TP-TK-06 |
| HOAT_DONG | TP-TK-02, TP-TK-05, TP-TK-07 |
| TAM_KHOA | TP-TK-03, TP-TK-04 |

**Dữ liệu test cần:**
- ≥1 tài khoản mới tạo chưa kích hoạt (cho TP-TK-01)
- ≥1 tài khoản active (cho TP-TK-02, TP-TK-05)
- Tài khoản QTHT riêng biệt (cho TP-TK-07 — verify giới hạn 1 phiên)

## Ghi chú thực thi

- **TP-TK-02 (5 lần sai):** Verify chặn ngay lần 6, không cho thử tiếp dù đúng password.
- **TP-TK-03 (auto unlock):** Nếu không mock time được, đánh dấu P2 hoặc verify qua cấu hình DB.
- **TP-TK-06 (hết hạn 7 ngày):** Similar — cần time control hoặc SQL backdating.
- **TP-TK-07 (session limit):**
  - User thường: login phiên 1/2/3 (OK) → phiên 4 → verify phiên 1 bị kick.
  - QTHT: login phiên 1 (OK) → phiên 2 → verify phiên 1 bị kick ngay.
- **Integration với MailHog:** Link kích hoạt (TP-TK-01) và link reset (liên quan QT-024) verify qua MailHog — xem [§9.1](../test-strategy.md#9-môi-trường-kiểm-thử).
